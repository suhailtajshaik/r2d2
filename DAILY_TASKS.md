# R2D2 Daily Tasks

Things R2D2 should do every day (or every session) to stay sharp and useful.

---

## Every Session

- [ ] Read latest memory files — know what happened recently
- [ ] Check brain repo for staleness — are any files outdated?
- [ ] Update `memory/YYYY-MM-DD.md` with session notes
- [ ] Sync workspace → brain repo (run `sync.sh`) before session ends

## Daily

- [ ] Web search for updates on active projects' dependencies (Vapi, Supabase, Bun, etc.)
- [ ] Check if any project containers are down: `docker ps -a`
- [ ] Update Notion session log with today's work
- [ ] Review `vps/state.md` — does it match actual `docker ps` output?

## Weekly

- [ ] Review `memory/operating-rules.md` — still accurate?
- [ ] Check GitHub repos for stale branches or open issues
- [ ] Research one topic from Suhail's research-links.md that hasn't been explored yet
- [ ] Update `vps/github-remotes.md` with any version bumps
- [ ] Clean up old memory files — archive anything >30 days old that's been captured in MEMORY.md

## When Suhail Starts a New Project

- [ ] Create Notion page for the project
- [ ] Add to `vps/github-remotes.md`
- [ ] Add to project priority list in `memory/operating-rules.md`
- [ ] Create `memory/research-{project}.md` with initial research
- [ ] Set up Docker + nginx routing if it needs to be deployed

## When Suhail Gives Feedback

- [ ] Save to `memory/operating-rules.md` or relevant memory file
- [ ] Update skills if the feedback applies to a skill
- [ ] Acknowledge and confirm the change so Suhail doesn't repeat himself
